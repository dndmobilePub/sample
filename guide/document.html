SCSS 가이드

CSS 방법론
OOCSS(Object Oriented CSS)
  • OOCSS(Object Oriented CSS)는 CSS를 모듈(module)방식으로 작성하여 중복을 줄이는 방식의 방법론
  • OOCSS는 가장 많이 사용되는 방법론으로 구조와 스타일을 분리하여 작성합니다.
    중복되는 디자인 요소를 따로 빼내어 작성하기 때문에 반복적으로 사용가능합니다. 또한 코드의 재사용성이 높아져 적은 코드량으로 최적의 성능을 보입니다.
  • 다중 클래스 사용으로 인해 코드의 가독성이 떨어질 수 있으나 공통된 부분을 찾아 어디서나 재활용 할수 있다는 장점이 있기 때문에 OOCSS 방법론에 따라 작성합니다

  • 속성값은 반드시 변수로 적용되도록 작성한다. (_variables.scss 에 선언된 변수값을 참고)

    WSG에 따라 필요시 _variables.scss 에 선언된 변수값을 수정/추가 할 수 있다.

  • 엘리먼트 상위에 선언되는 wrap가 있는 클래스는 카멜 스타일로 선언(ex. classWrap) 그외 클래스는 예약어-클래스명, 또는 클래스성격-클래스명 등의 형태인 (-)하이픈으로 연겨된 클래스명을 갖도록 작성한다 
                                                        /* 카멜선언 */
                                                    .btnWrap {}
                                                    .tblWrap {}
                                                    .lstWrap {}
                                                    
                                                    /* (-) 하이픈선언 */
                                                    .btn {
                                                        &-style {}
                                                    }
                                                    .cont {
                                                        &-desc {}
                                                    }
  • scss 속성 선언시 모든 속성은 소문자로 작성되어야 하며, 선언된 속성과 속성 사이는 빈칸으로 구분하고 속성과 속성값 사이에 빈칸은 적용하지 않는다. 
                                                        /* 잘못된 사용 예
                                                    1. 속성과 속성값 사이 빈칸 적용
                                                    2. 속성에 대문자 사용
                                                    */
                                                    .example {display: block; Background: $color-white; COLOR: $color-black;}
                                                    
                                                    /* 올바른 사용 예 */
                                                    .example {display:block; background:$color-white; color:$color-black;}
  • 페이지내 style 사용은 금하며, 예외적으로 input 또는 팝업(레이어팝업 포함) 사이즈 지정이 필요한 경우 style="width" 사용할 수 있다.
  • 문서내 margin/padding 사용이 필요할 경우 해당 요소의 클래스내에서 처리하며, 아래 예시와 같이 inner style 또는 공통클래스(ex. marT15, padL20 등)를 만들어 사용하지 않는다 
                                                        <!-- 잘못된 사용 예 -->
                                                    <p style="margin-top:20px; padding-bottom:15px;">사용금지</p>
                                                    <p class="marT20 padL20">사용금지</p>
                                                    
                                                    <!-- 올바른 사용 예 -->
                                                    p.sampleBox {margin-top:20px; padding-bottom:15px;}
                                                    <p class="sampleBox">inner style 또는 공통클래스(ex. marT15, padL20 등)를 만들어 사용하지 않는다.</p>
  • 속성값으로 한글폰트 및 이미지 URL 표기시 작은 따옴표{''}를 사용하며, 모든 속성의 마지막에는 세미콜론(;)을 작성하도록 한다. 
                                                    div {font-family:'맑은고딕', '돋움'; background:url(/front/img/temp.gif);} /* 속성 마지막에는 반드시 (;) 작성. */

JS 가이드

JS 가이드
JQUERY 선언
  • jquery 버전은 항상 최신 버전(jQuery 3.x 기준)을 사용하며, 필요시 하위 버전과의 호환성을 위해 jQuery Migrate를 함께 사용할 수 있다. 
                                                        
                                                    <script type="text/javascript" src="./asset/js/plugin/lib.jquery-3.7.1.min.js"></script>
                                                    
                                                    <script type="text/javascript" src="./com/js/jquery-migrate-3.4.0.min.js"></script>
                                                    
                                                    <script type="text/javascript" src="./com/js/jquery-ui-1.13.2.js"></script>
JS 파일명 작성 규칙
  • js 파일은 기본적으로 [./asset/js/] 폴더 내에 포함된다.
  • 각 파일마다 성격에 따라 지정된 접두어로 시작되며, [.]로 파일명 네임을 구분하도록 한다.
  • jquery library/plugin 을 사용할 경우 파일명 앞에 [lib.*.js], [pg.*.js]의 형태로 작성되어야 한다. 
                                                        <script type="text/javascript" src="./asset/js/plugin/lib.jquery-3.7.1.min.js"></script>
                                                    <script type="text/javascript" src="./asset/js/plugin/pg.cropper.js"></script>
  • 퍼블리싱팀에서 작성하는 js 파일은 [ui.*.js]의 형태로 작성되어야 한다. 
                                                        <script type="text/javascript" src="./asset/js/ui.component.js"></script>
